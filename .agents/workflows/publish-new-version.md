# Описание
Алгоритм правильной публикации новой версии плагина.

# Инструкция
1. **Синхронизация GitHub:** Сначала обязательно добавь все файлы (`git add .`), сделай коммит и отправь изменения в удаленный репозиторий GitHub (`git push`).
2. **Публикация в LobeHub:** После обновления GitHub, вызови скилл `/lobehub-mcp-publisher` для корректной публикации плагина в маркетплейс (команда `lhm plugin publish --dir .`).
